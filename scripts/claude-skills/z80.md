# SKILL: z80 — Zilog Z80 sur Mega Drive (contexte son)

## Rôle sur MD
Le Z80 est le processeur audio. Il tourne à 3.58 MHz (NTSC) / 3.55 MHz (PAL).
Il gère le YM2612 (FM) et le SN76489 (PSG) via des drivers son (GEMS, SMPS, XGM...).

## Espace mémoire Z80
```
$0000-$1FFF   Z80 RAM (8KB) — programme + données son
$2000-$3FFF   miroir RAM Z80
$4000-$4001   YM2612 port 0 (regs 0x00-0x9F)
$4002-$4003   YM2612 port 1 (regs 0x00-0x9F, canal 4-6)
$6000-$6000   bank switch register (accès ROM 68k)
$7F11         SN76489 PSG data port
$8000-$FFFF   window 32KB sur ROM 68k (bank switching via $6000)
```

## Bus arbitration (côté 68k)
```c
// Arrêter le Z80 (avant d'accéder à sa RAM)
*((volatile u16*)0xA11100) = 0x100;      // Z80 BUS REQUEST
while (*((volatile u16*)0xA11100) & 0x100); // attendre BUSACK

// Accéder à la Z80 RAM
// ... lecture/écriture $A00000-$A01FFF ...

// Rendre le bus au Z80
*((volatile u16*)0xA11100) = 0x000;

// Reset Z80
*((volatile u16*)0xA11200) = 0x000;  // assert RESET
// ... délai ...
*((volatile u16*)0xA11200) = 0x100;  // release RESET
```

## Instructions Z80 clés pour audio
```asm
; Écriture YM2612
ld   bc, $4000      ; port 0
out  (c), a         ; écrire registre
; ou
ld   ($4000), a     ; écrire index reg
ld   ($4001), a     ; écrire data

; Écriture PSG
ld   a, data
ld   ($7F11), a

; Accès ROM via bank switching
ld   a, bank_bits   ; 9 bits de bank (bits 0-8)
; écrire bit par bit dans $6000
ld   bc, $6000
rept 9
  ld   a, bank_bits >> iteration
  and  1
  out  (c), a
endr
```

## Drivers son courants sur MD

### SMPS (Sonic Music/Sound Program System)
- Utilisé par la plupart des jeux Sega 1st party
- Format custom : tracks, patterns, DAC

### GEMS (Genesis Editor for Music and Sound effects)
- Très répandu chez développeurs tiers occidental
- Known pour bloquer les interruptions longtemps (problèmes timing)

### XGM / XGM2 (SGDK)
- Driver moderne de Stef D pour SGDK
- XGM2 : supporte jusqu'à 8 canaux FM + 4 PSG + 4 PCM Z80 RAM
- Initié par `XGM2_init()`, musique via `XGM2_playMusic()`

## Breakpoints Z80 dans BlastEm
```
(blastem debugger) zb $0100    ; breakpoint Z80 à $0100
(blastem debugger) zp $40      ; print registre Z80 $40 (YM2612 status)
```

## Pièges communs
- Ne jamais accéder à la Z80 RAM sans avoir le bus
- Pas de DMA possible pendant que le Z80 tourne (BUSREQ sinon)
- Le Z80 n'a accès qu'à 32KB de ROM à la fois (bank switching nécessaire si ROM > 32KB)
- SN76489 : write-only, registres latched, initialiser proprement (atténuer tous canaux)
